LSL：Lab Streaming Layer

相关名词介绍

Sample: 对一台设备的所有通道进行一次测量

Chunk: 样本的集合发送

MetaData: XML记录关于流的信息，类似于文件头

Stream: 设备的采样数据与元数据的组合称为流，流的采样率可以相同可以不同，可以来自一个通道也可以来自很多通道，流内数据必须为相同类型

Stream Outlet: 用于提供时间序列数据流，数据被逐样本或逐块推送到出口；通过outlet可以使数据流被网络中的一批主机可见，主机可以通过解析函数找到outlet以及订阅

Stream Inlet: 用于从连接的outlet中接收时间序列数据，允许从流中检索采样数据，inlet还可以获得元数据（以XML形式或者通过小型的内嵌DOM接口）

Resolver: LSL提供函数来解析数据流（根据以内容为基础的查询），查找服务并不依赖于外部设备，旨在简化数据采集网络

Built-in clock: 允许对传输的样本进行时间戳，以便它们可以相互同步

Time Synchronization: LSL具有的内嵌时间同步设备，详细内容，NTP

File Format: 传输API本身不提供特定的文件格式，但提供的记录程序（默认）将数据记录为XDF格式，XDF与LSL同时设计，支持LSL的完整功能集（包括多流，任意大小的XML头，所有的样本格式以及时间同步信息）

具体说明

Outlets: 一个出口可能使用ipv4，也可能使用ipv6，也可能同时使用，lsl::stream_outlet

UDP Service Discovery: 一个出口创建多个UDP socket，用于监听查询。这样的查询都是用于查找感兴趣的数据流，出口将每个入站查询与其元数据匹配，一旦匹配则响应，重复的查询将被缓存；特别地，outlets监听组播端口(16571)，用于接收组播和多播请求，并且为此创建了多个socket，用于监听不同的多播地址，这些socket采用异步I/O与单线程，**虽然这听起来仍然花销巨大，但请注意线程在大多数时候都处于等待状态**；另外，outlets创建了一个单播的UDP socket用于在**free port**（16572-16604）上监听查询，这个socket不仅用于处理内容查询也用于响应时间估计数据包（由入口发出）

TCP Data Transmission: 另外每个出口创建一个TCP服务端socket通过**free port**（16572-16604）接收实际的数据传输请求,传输请求包括两种：请求数据流和请求元数据，TCP服务器也可以用于处理服务发现，但是因为低效不被用于这个目的

Resolve Operations: 当一个客户端创建了一个Stream Solver来解析实验室网络中具有所需属性的流时，它会发出解析操作，解析操作会临时打开一系列UDP sockets，通过这些socket，查询被发送到激活的地址，它同样会开启一个UDP socket来接收结果包，解析操作会在一个短暂的时间内**向每个通道**发送一个数据包，并等待响应，然后发送下一个突发，知道找到服务或超时

Inlets: Inlet在被创建时是默认没有任何socket的，创建socket的情况有以下几种：1）调用lsl::stream_inlet::pull_sample()或者lsl::stream_inlet::open_stream()以开启数据流的传输，此时将会建立一个TCP连接，  2）调用lsl::stream_inlet::info()，    3）当调用lsl::stream_inlet::time_correction()用于估计**时间校正值**时，将打开一个UDP socket，与出口的UDP单播端口**交换一系列简短的数据包**，

Clock synchronization: 本地时钟函数lsl_local_clock()，查询时间偏移函数lsl_time_correction_ex()，获取加载库的版本的函数lsl_library_version()
